## Architecture at a Glance

**Pattern**: Clean Architecture + DDD + CQRS + Microservices

```
src/
├── Domain/                    # CORE - zero external dependencies (enforced by tests)
├── Infrastructure/            # Data access - depends only on Domain
├── GrpcServer.Contracts/      # Shared CQRS command DTOs
├── GrpcServer/                # gRPC service - independent from WebApi
├── WebApi/                    # REST API via FastEndpoints
├── IdentityApi/               # JWT authentication service
├── WebClient/                 # Blazor WASM frontend
└── WebApi.Client/             # Auto-generated Kiota API client

test/
├── Architecture.Tests/        # 25+ NetArchTest rules - MUST always pass
├── WebApi.Unit.Tests/         # NUnit + FakeItEasy
└── WebApi.Integration.Tests/  # Alba - requires running database
```

### Dependency Flow (strictly enforced)

```
WebApi / GrpcServer / IdentityApi / WebClient
        ↓
   Infrastructure
        ↓
      Domain          ← no outward dependencies allowed
```

**GrpcServer is NOT a dependency of WebApi.** They are two independent service implementations, both accessing Infrastructure separately.

## Critical Rules (Enforced by Architecture.Tests)

Breaking any of these will cause architecture tests to fail:

1. **Domain purity** - Domain has zero references to EF Core, Dapper, Npgsql, or any external library
2. **Dependency direction** - Infrastructure → Domain only, never reversed
3. **Sealed entities** - All domain entities are `sealed class` inheriting `BaseEntity`
4. **Factory methods** - Entities are created via `static Create(...)` methods, never `new` directly from outside
5. **Naming conventions**:
   - DTOs → end with `Dto`
   - Commands → end with `Command`
   - Handlers → end with `Handler`
   - Repository interfaces → start with `I`, end with `Repository`
   - FastEndpoints classes → named exactly `Endpoint`
6. **Soft deletes** - Never hard delete; use `Entity.Remove(obj)` which sets `Active = false`

## Data Access Patterns

### Dual Strategy (important - both coexist)

**Reads → EF Core (via `IApplicationDbContext`)**
```csharp
// Injected in WebApi endpoints
public class Endpoint(IApplicationDbContext dbContext) : Endpoint<Request, Response>
{
    // Use LINQ, tracked entities, navigation properties
    var item = await dbContext.Assignments!.FindAsync(req.Id, ct);
}
```

**Writes → Two approaches depending on context:**

Option A - Direct EF Core (simple WebApi endpoints):
```csharp
await dbContext.Assignments!.AddAsync(newItem, ct);
await dbContext.SaveChangesAsync(ct);
```

Option B - Dapper via Unit of Work (GrpcServer handlers, transactional operations):
```csharp
await unitOfWork.BeginTransactionAsync();
var repo = unitOfWork.GetRepository<Assignment>();
await repo.AddAsync(entity);
await unitOfWork.CommitAsync(ct);
// Always catch and RollbackAsync on exception
```

### Unit of Work Interface
`src/Domain/UoW/IUnitOfWork.cs`
- `GetRepository<T>()` where T : BaseEntity
- `BeginTransactionAsync()` / `CommitAsync()` / `RollbackAsync()`

### EF Core Global Query Filters
All entities have a global filter `WHERE Active = true`. Soft-deleted records are invisible to EF Core unless you explicitly use `.IgnoreQueryFilters()`.

## Entity Pattern (Domain Layer)

Reference file: `src/Domain/Entities/Assignment.cs`

```csharp
namespace Domain.Entities;

public sealed class Assignment : BaseEntity   // sealed is REQUIRED
{
    public string? Name { get; set; }
    public Guid ProjectId { get; set; }
    public Project? Project { get; set; }     // navigation property

    public static Assignment Create(string? name, Guid projectId, Guid id = default)
    {
        return new Assignment
        {
            Id = BaseEntity.GetNewId(id),     // UUID v7
            CreatedAt = DateTime.UtcNow,
            Active = true,
            Name = name,
            ProjectId = projectId
        };
    }

    public static void Update(Assignment obj, string? name, Guid projectId)
    {
        obj.Name = name;
        obj.ProjectId = projectId;
        obj.UpdatedAt = DateTime.UtcNow;
    }

    public static void Remove(Assignment obj)   // soft delete
    {
        obj.Active = false;
        obj.UpdatedAt = DateTime.UtcNow;
        obj.DeletedAt = DateTime.UtcNow;
    }
}
```

`BaseEntity` provides: `Id` (Guid), `CreatedAt`, `UpdatedAt?`, `DeletedAt?`, `Active`.

## FastEndpoints Pattern (WebApi)

Reference folder: `src/WebApi/Endpoints/Assignment/`

Each operation lives in its own subfolder with two files: `Endpoint.cs` and `Models.cs`.

```csharp
// Endpoint.cs
public class Endpoint(IApplicationDbContext dbContext) : Endpoint<Request, Response>
{
    public override void Configure()
    {
        Post("/api/assignment");
        Description(x => x.WithTags("Assignments"));
        AllowAnonymous();                              // or RequireAuthorization()
        Summary(s =>
        {
            s.Summary = "Create a new assignment";
            s.Description = "Creates and persists a new assignment.";
        });
    }

    public override async Task HandleAsync(Request req, CancellationToken ct)
    {
        // Validation
        if (dbContext.Assignments!.Any(x => x.Id == req.Id))
            AddError(r => r.Id, "this Id is already in use!");
        ThrowIfAnyErrors();

        // Business logic
        var entity = Assignment.Create(req.Name, req.ProjectId);
        await dbContext.Assignments!.AddAsync(entity, ct);
        await dbContext.SaveChangesAsync(ct);

        Response.Assignment = entity.MapToDto();
        await SendOkAsync(Response, ct);
    }
}
```

```csharp
// Models.cs
public record Request
{
    public Guid Id { get; init; }
    public string? Name { get; init; }
    public Guid ProjectId { get; init; }
}

public record Response
{
    public AssignmentDto? Assignment { get; set; }
}
```

## GrpcServer Handler Pattern

Reference folder: `src/GrpcServer/Handlers/Assignment/`

```csharp
public sealed class CreateAssignmentHandler(IUnitOfWork unitOfWork, ILogger<CreateAssignmentHandler> logger)
    : ICommandHandler<CreateAssignmentCommand, CreateAssignmentResult>
{
    public async Task<CreateAssignmentResult> ExecuteAsync(CreateAssignmentCommand command, CancellationToken ct)
    {
        try
        {
            await unitOfWork.BeginTransactionAsync();
            var repo = unitOfWork.GetRepository<Assignment>();
            var entity = Assignment.Create(command.Name, command.ProjectId);
            await repo.AddAsync(entity);
            await unitOfWork.CommitAsync(ct);

            return new CreateAssignmentResult { Success = true, Assignment = entity.MapToDto() };
        }
        catch (Exception ex)
        {
            logger.LogError(ex, "Error creating assignment");
            await unitOfWork.RollbackAsync(ct);
            throw;
        }
    }
}
```

## Object Mapping (Mapperly)

Mappers are compile-time source-generated. Never use AutoMapper or manual mapping.

Reference file: `src/Infrastructure/` or check for `EntityToDtoMapper.cs` / `Mapper.cs` per entity area.

```csharp
[Mapper]
public static partial class EntityToDtoMapper
{
    public static partial AssignmentDto MapToDto(this Assignment entity);

    [MapperIgnoreSource(nameof(User.Password))]
    public static partial UserDto MapToDto(this User entity);
}
```

Usage: `entity.MapToDto()` - extension method, no DI needed.

## Adding a New Entity (Checklist)

Follow this exact order to avoid architecture test failures:

1. `src/Domain/Entities/NewEntity.cs` - sealed class, BaseEntity, Create/Update/Remove
2. `src/Domain/Repositories/INewEntityRepository.cs` - extends `IRepository<NewEntity>`
3. `src/Infrastructure/Common/Mappings/NewEntityMap.cs` - `IEntityTypeConfiguration<NewEntity>`
4. `src/Infrastructure/Common/Context/ApplicationDbContext.cs` - add `DbSet<NewEntity>`
5. Run migration:
   ```bash
   dotnet ef migrations add AddNewEntity -p ./src/Infrastructure -s ./src/WebApi -c ApplicationDbContext
   dotnet ef migrations script --output ./docker-entrypoint-initdb.d/001-database-dump-ddl.sql --idempotent -p ./src/Infrastructure -s ./src/WebApi -c ApplicationDbContext
   ```
6. `src/GrpcServer.Contracts/Commands/NewEntity/` - Create/Update/Remove commands
7. `src/GrpcServer/Handlers/NewEntity/` - handlers using Unit of Work
8. `src/WebApi/Endpoints/NewEntity/` - 5 endpoints (Create, GetById, List, Update, Remove)
9. Add Mapperly extension methods for NewEntity → NewEntityDto
10. `dotnet test test/Architecture.Tests` - must pass before proceeding

## Common Pitfalls

| Mistake | Consequence | Fix |
|---|---|---|
| Adding EF Core reference to Domain | Architecture test failure | Move data access to Infrastructure |
| Non-sealed entity class | Architecture test failure | Add `sealed` keyword |
| Naming class anything other than `Endpoint` | Architecture test failure | Rename to `Endpoint` |
| Hard-deleting entities | Breaks soft-delete invariant | Use `Entity.Remove(obj)` |
| Using `new Guid()` for entity IDs | Non-sortable UUID v4 | Use `BaseEntity.GetNewId(id)` |
| Manual object mapping without Mapperly | Runtime reflection overhead | Add `[Mapper]` partial methods |
| Running migrations with wrong startup project | Migration fails | Always use `-s ./src/WebApi` |
| Referencing GrpcServer from WebApi | Breaks independence | They must stay separate |
| Forgetting `ThrowIfAnyErrors()` after `AddError()` | Validation never fires | Call before business logic |
